# VibeStack 데이터 품질 & 콘텐츠 가이드라인

이 문서는 VibeStack에 등록되는 모든 도구(Tool) 및 콘텐츠의 품질과 일관성을 유지하기 위한 규칙을 정의합니다. 관리자와 기여자는 이 가이드를 준수해야 합니다.

## 1. 이미지 규격 (Images)

사용자에게 전문적인 인상을 주기 위해 고품질의 이미지를 사용합니다.

### 로고 (Logo)
- **비율**: 1:1 (정사각형)
- **최소 크기**: 512x512 px
- **권장 포맷**: SVG (최우선), PNG (투명 배경)
- **주의사항**:
  - 배경이 투명하지 않은 경우, 모서리가 둥글게 처리된(Rounded) 이미지를 사용하지 마십시오. CSS로 처리합니다.
  - 저해상도, 픽셀이 깨지는 이미지는 사용할 수 없습니다.

### 스크린샷 / 대표 이미지
- **비율**: 16:9 권장
- **최소 크기**: 1280x720 px
- **권장 포맷**: WebP, JPG (압축률 80% 이상)
- **내용**: 도구의 핵심 기능을 보여주는 실제 인터페이스 화면. 마케팅용 텍스트가 과도하게 포함된 배너는 지양합니다.

---

## 2. 텍스트 스타일 (Text Style)

정보는 객관적이고 간결하게 전달해야 합니다.

### 도구 이름 (Name)
- 공식 명칭을 사용합니다. (예: "GitHub Copilot" O, "Github copilot" X)
- 불필요한 수식어를 제외합니다. (예: "Best AI Tool" 등 제외)

### 한 줄 소개 (Tagline / Description)
- **길이**: 국문 30자 / 영문 60자 이내 권장
- **톤앤매너**: "~하는 도구", "~를 위한 AI" 형태로 명사형 종결 또는 간결한 문장.
- **금지**: "최고의", "놀라운", "세계 최초" 등의 주관적이고 과장된 표현.객관적 사실 위주로 작성.

### 상세 설명 (Long Description)
- **구조**:
  1. **개요**: 무엇을 하는 도구인가?
  2. **핵심 기능**: 주요 기능 3~5가지를 불렛 포인트로 나열.
  3. **대상 사용자**: 누가 쓰면 좋은가?
- MarkDown 문법을 활용하여 가독성을 높입니다.

## 3. 분류 체계 (Taxonomy)

### 카테고리 (Category)
- 기존에 정의된 표준 카테고리 중 하나를 선택합니다.
- 새로운 카테고리가 필요한 경우, 개발팀과 협의 후 추가합니다.
- **주요 카테고리**:
  - `Coding`: 개발 보조, 코드 생성
  - `Design`: 이미지 생성, UI 디자인
  - `Productivity`: 업무 자동화, 문서 요약
  - `Marketing`: 카피라이팅, SEO
  - `Video`: 영상 편집, 생성

### 태그 (Tags)
- **형식**: kebab-case (소문자, 하이픈 연결) 권장. (예: `open-source`, `free-tier`)
- 도구의 특성(무료 여부, 플랫폼, 기술 스택)을 나타내는 키워드 위주로 입력.

---

## 4. 메타데이터 (Metadata)

- **Website URL**: 추적 매개변수(UTM 등)가 없는 깨끗한 메인 페이지 URL.
- **Affiliate URL**: 제휴 링크가 있는 경우 별도 필드에 입력.
- **Pricing**: `Free`, `Freemium`, `Paid`, `Contact` 중 정확한 모델 선택.
