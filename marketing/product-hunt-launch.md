## 2. 첫 코멘트 (Maker's Comment) ⭐ 가장 중요

PH에서 트래픽/투표를 좌우하는 것은 **첫 코멘트**. 게시 후 5분 안에 본인이 댓글 달아야 함. 너무 마케팅스럽지 않게, "왜 만들었나"와 "어떻게 다른가"에 집중.

```
Hey Product Hunt 👋

I'm David, maker of VibeStack.

I built this because picking AI tools had become its own full-time job:

1. Google "best AI [thing]" → 10 SEO listicles ranking whoever pays the
   most
2. Open 20 tabs comparing pricing pages
3. Forget which one had the feature I actually needed
4. Give up and stay on whatever I was already using

VibeStack tries to fix that with three things:

→ **5-question quiz → curated stack.** Answer 5 questions about your
  role, budget, and workflow, and get a hand-picked combo of 3–5 tools
  that work together. Skip the comparison spreadsheet. (60 seconds.)

→ **Stacks, not just lists.** Pre-built combos like "10x Engineer Stack"
  ($40+/mo, 2 tools, 4-step workflow) or "Product Designer Stack" — the
  way real builders actually combine tools, not isolated rankings.

→ **Vetted monthly, no affiliate ranks.** 30+ tools right now, rechecked
  every month for pricing/feature drift. Rankings are based on real-use
  validation, not who emails me. When a placement is sponsored, it'll be
  labeled clearly (you'll see "PROMOTED" tags on the site).

What's NOT here yet (being honest):
- Bigger catalog — staying at 30+ deliberately until quality bar holds
- More expert reviews per tool (some still say "Full review coming soon")
- Price change alerts (Pro feature, not shipped yet)

I'd love feedback on:
1. The 5-question quiz — does the result actually feel like *your* stack?
2. The "stack" concept — does it match how you pick tools, or do you
   prefer the flat-list approach?
3. Any tool I'm missing that's been a game-changer for you?

Free forever for browsing. Pro plan ($9/mo) is for saving custom stacks,
team workspaces, and price-change alerts when those ship.

Thanks for checking it out 🙏
```

**커스터마이즈할 부분:**
- `$9/mo` → 실제 Pro 가격으로 교체 (pricing 페이지 확인 후)
- `$40+/mo`, `2 tools, 4-step workflow` → /stack/10x-engineer 실제 값 (캡처 보니 정확)
- 마지막 질문 3개 → 본인이 진짜 듣고 싶은 피드백으로
- 이모지는 1~2개로 줄여도 됨 (취향)

## 3. 예상 Q&A — 미리 답 준비

PH 댓글에 자주 나오는 질문들. 본인이 시드로 댓글 달아도 되고, 누가 물으면 빠르게 답할 수 있게 미리 준비.

### Q1: "How is this different from [Futurepedia / There's An AI For That / SaaSHub]?"
```
Great question. The big AI directories optimize for SEO (catching every
"best AI X" search), so they list 10,000+ tools with shallow data.

VibeStack is the opposite — small, curated, and focused on *combinations*
(stacks) instead of just lists. Closer to a Pinterest board for AI
workflows than a Yellow Pages.

If you want the longest list, those sites win. If you want "what should
I actually use together?" — that's the gap I'm trying to fill.
```

### Q2: "How do you make money?"
```
Right now: nothing. Site's been live for ~3 months as a side project.

Plan: Pro plan for power features (saved stacks, price change alerts,
private team workspaces) + optional sponsored placements that will be
*clearly labeled as such* and won't affect organic rankings. No affiliate
links hidden in reviews. If you ever see one without disclosure, call me
out publicly.
```

### Q3: "Can I submit my tool?"
```
Yes! https://usevibestack.com/submit-tool

I personally review every submission. Bar is "would I actually recommend
this to a friend building something?" — not "does this tool exist." So
expect feedback if I'm not adding it (and feel free to push back).
```

### Q4: "Why 'vibe' stack?"
```
"Vibe coding" — the Karpathy-coined term for building software with AI
assistants doing the heavy lifting. The "vibe" is the high-level intent;
the AI translates it to code. VibeStack is the toolset that makes vibe
coding actually work.
```

### Q5: "Open source?"
```
The data (tools, categories) is closed-source for now — moderation takes
real time and I want to keep quality high. If there's demand, I'll open
up an API for non-commercial use. The frontend is closed too but I'm open
to contributions via the submit-tool flow.
```

### Q6: "Mobile app?"
```
Not yet. The site is mobile-responsive and PWA-installable, which covers
most reading. A native app only makes sense if usage patterns suggest it
— I'd rather nail the web experience first.
```

## 5. 런치 D-Day 타임라인 (PST 기준)

PH는 **00:01 PST에 새 날짜 시작**. KST 17:01에 게재되는 셈. 한국 시간으로 일 시작이라 편함.

| Time (PST) | Time (KST) | Action |
|---|---|---|
| 11:50 PM (전날) | 16:50 | PH 대시보드 열어두고 대기. 친구/팀에 "5분 전" 알림 |
| 12:01 AM | 17:01 | 게재 자동 발행. 1분 내 본인이 maker comment 작성 |
| 12:05 ~ 1:00 AM | 17:05 ~ 18:00 | 사전 동의받은 30~100명에게 링크 푸시 (DM, 단톡, 이메일) |
| 1:00 ~ 6:00 AM | 18:00 ~ 23:00 | 댓글 100% 응답. 5분 안에 답글. 질문 없으면 본인이 시드 질문 댓글 (다른 PH 친구가 달아주면 더 자연스러움) |
| 6:00 AM | 23:00 | X에 런치 트윗 (스크린샷 + 링크). "VibeStack is live on PH today" |
| 9:00 AM ~ 6:00 PM | 02:00 ~ 11:00 (다음날) | 미국 시간대 활성 구간. 댓글 응답 + Indie Hackers / r/SideProject에 "We're live on PH" 글 (광고스럽지 않게) |
| 11:59 PM | 16:59 | 마감. 최종 순위 확인 |

**핵심 룰:**
- **댓글에 5분 안에 답하기** — PH 알고리즘이 maker engagement를 가장 크게 봄
- **"투표해줘" 절대 금지** — "feedback 줘", "써보고 알려줘" 톤만
- **부정적 댓글에 방어적이지 말기** — 동의/감사 + 구체적 follow-up 질문으로 대화 이어가기

## 7. 안 하면 안 되는 것 vs 하면 좋은 것

### 🚫 절대 금지 (계정 정지 사유)
- 가짜 계정으로 투표 조작
- 댓글에 "upvote please" 류 직접 부탁
- 동시에 같은 사람들에게 모두 같은 시간에 투표 요청 (PH가 IP/패턴 감지)
- 이미 다른 곳에서 출시한 제품을 "신제품"처럼 등록 (이건 OK, 재출시도 허용되지만 정직하게 표시)

### ✅ 효과 검증된 것
- Maker가 모든 댓글에 5분 내 답글
- 첫 코멘트가 솔직하고 길고 구체적 (단순 "Check it out!" X)
- 갤러리 이미지가 실제 UI 스크린샷 위주 (모형/마케팅 컷보다)
- 런치 후 일주일간 maker 활동 유지
